# 配镜推荐

## 概览

把这个 skill 视为“实用配镜决策流程”，不是医疗诊断工具。按这个顺序工作：

1. 判断验光数据能不能用于推荐
2. 输出用户可以直接拿去买框的镜框约束
3. 推荐折射率和镜片档位

不要先报品牌名。如果镜框方向错了，或者验光数据本身偏弱，先把问题讲清楚。
不要把“字段看起来完整”误当成“处方足够可信”。普通单光购物建议可以保守给，高端定制建议必须更严格。

公式、品牌示例和报告模板放在 [references/selection-reference.md](references/selection-reference.md)。

如果你需要直接拿来和用户对话，读取 [references/conversation-templates.md](references/conversation-templates.md)：
- 标准问诊开场
- 文字输入路径
- 图片上传路径
- 缺信息时的追问顺序
- `Green / Yellow / Red` 三类输出模板
- 完整示例回答

如果你要验证 skill 是否跑偏，读取 [references/test-cases.md](references/test-cases.md)：
- 覆盖文字输入、图片上传、已选镜框、电脑验光、防蓝光、高端定制和 `Red-strict`
- 每组都包含预期行为和禁止行为

如果用户追问这些主题，额外读取 [references/linked-sources.md](references/linked-sources.md)：
- 镜架三姿态、前倾角、镜眼距、面弯
- 验光流程、点瞳高、单光是否需要瞳高
- 蔡司泽锐 / 智锐等高端镜片的验配要求
- 防蓝光、数码眼疲劳、屏幕用眼建议
- 膜层、防污和蔡司定制片辨别

## 适用范围

最适合：
- 成人单光近视配镜
- 近视合并散光配镜
- 想为日常佩戴选择镜框和镜片的用户
- 想先拿到实用购物约束再下单的用户

不适合做精确高端推荐的场景：
- 渐进、多焦点、抗疲劳、老花、防控类镜片
- 隐形眼镜
- 明显双眼视、视功能异常相关问题
- 术后、病理性或眼病相关场景
- 儿童青少年近视防控、弱视或斜视相关场景
- 突然视力变化、眼痛、闪光感、视野缺损、外伤或疑似眼病

遇到这些情况时：
- 只给原则级建议
- 明确建议专业验光或专项验配
- 如果出现急性症状，优先建议尽快就医，不继续做购物建议

## 工作流程

### 入口路径

这个 skill 支持两条入口路径，但后续会收束到同一套判断流程：

1. **文字路径**
- 用户直接输入 `OD / OS / PD / 用途 / 镜框参数`
- 直接进入“标准化输入”

2. **图片路径**
- 用户上传验光单、眼镜店处方单、医院打印单、电脑验光单图片
- 先从图片中提取关键字段
- 再让用户确认字段
- 然后进入同一套“标准化输入”

注意：
- `入口路径` 不等于 `数据可信度`
- 就算是文字路径，只要用户明确说这组数字来自电脑验光，也要按偏弱数据处理，不能自动视为最终配镜处方

如果是图片路径，额外读取：
- [references/selection-reference.md](references/selection-reference.md) 里的“图片验光单提取规则”
- [references/conversation-templates.md](references/conversation-templates.md) 里的“图片上传路径模板”

### 第一步：标准化输入

先收最少可用信息：
- `OD`: `SPH / CYL x AXIS`
- `OS`: `SPH / CYL x AXIS`
- `PD`：优先单眼瞳距；没有时接受总瞳距
- 主要用途：看远 / 室内电脑 / 混合使用
- 是否在意厚薄
- 预算区间

纯球镜的标准化规则：
- 如果某只眼只有球镜度数，且没有散光记录，可以先标准化为 `CYL 0.00`
- 只有在存在散光时，`AXIS` 才是硬必填项
- 不要因为纯球镜没有 `AXIS`，就把本来可用的文字处方误判为不完整

强烈建议补充：
- 当前眼镜的舒适度和适应反馈
- 现有镜框参数：`A`、`DBL`、镜腿长、片高
- 验光质量线索：红绿测试、JCC、试戴、瞳高

如果用户上传的是图片而不是文字：
- 优先提取右眼、左眼、散光、轴位、`PD`
- 区分“电脑验光结果”和“最终配镜处方”
- 如果字段模糊、裁切不全、手写不清，必须让用户二次确认
- 对无法辨认的字段，不要猜
- 用户确认图片识别结果，只代表字段抄录确认，不代表这张单子自动升级为最终配镜处方

如果用户只给“多少度”，先继续追这些缺口：
- 散光轴位
- `PD`
- 旧镜是否舒适
- 使用场景
- 是否已经选了镜框

需要快速开问时，直接套用 [references/conversation-templates.md](references/conversation-templates.md) 里的“标准问诊模板”。

如果用户先发的是验光单图片，直接套用同文件里的“图片上传路径模板”。

如果用户只会说“近视多少度”，先映射到 `SPH`，并明确说明数据还不完整。

如果用户用文字输入，但明确标注“这是电脑验光结果”“机器打出来的”：
- 不要因为字段看起来完整，就直接判成 `Green`
- 默认至少按偏保守的 `Yellow`，很多情况下应进 `Red`
- 可以继续给镜框方向和保守折射率范围，但不要直接做高端定制推荐

### 第二步：给验光数据设闸门

在给任何具体推荐前，先分级。分级时同时看两个维度：

1. **字段完整度**：`SPH / CYL / AXIS / PD / 用途 / 镜框参数` 是否够用
2. **处方可信度**：是否有主观验光、试戴、红绿测试、JCC、旧镜适应反馈；是否只是电脑验光

输出里可以合并成 `Green / Yellow / Red`，但心里必须先拆开判断：
- 字段完整但只有电脑验光：不是 `Green`
- 字段略少但来源可靠、旧镜反馈稳定：可以给普通单光的保守建议
- 任何高端定制判断，都要以处方可信度和定位数据为主，不只看字段完整

#### Green

满足大部分以下条件时，用 `Green`：
- 有完整 `SPH`，以及在存在散光时的 `CYL / AXIS`
- 有 PD
- 不只是电脑验光
- 有主观验光或试戴依据
- 用户能描述旧镜舒适度和佩戴表现

补充理解：
- 文字路径里，如果一只眼是纯球镜，另一只眼数据完整，且有 `PD + 用途 + 旧镜反馈`，通常已经足够进入普通单光配镜的“实用 Green”
- “实用 Green”只表示可以做普通单光购物决策，不自动等于“处方质量足够上高端定制”
- 高端定制还要额外看瞳高、镜框稳定性、镜架定位数据和验配质量

`Green` 可以：
- 给镜框约束
- 给折射率建议
- 给常规镜片档位建议
- 只有在镜框稳定、定位数据也足够好时，才给高端定制镜片建议

#### Yellow

数据不完整但还能做中等确定性建议时，用 `Yellow`：
- 只有总 PD
- 没有片高或瞳高
- 验光流程不清楚
- 旧镜反馈较弱

`Yellow` 可以：
- 给镜框范围建议
- 给折射率建议
- 给普通单光镜片档位建议

`Yellow` 不可以：
- 自信地推荐高端定制方案
- 用像“已经确定无疑”一样的语气说结论

#### Red

命中以下任一情况时，进入 `Red`：
- 只有电脑验光单
- 轴位缺失或明显可疑
- 多次验光结果冲突很大
- 有复视、严重视疲劳、斜视、融像异常等表现
- 明显屈光参差且没有适应史
- 用户要配渐进、防控或视功能类产品
- 儿童青少年近视防控、眼病、术后或急性症状

把 `Red` 再拆成两种处理：

1. **Red-strict**：有症状红旗、儿童防控、眼病/术后、渐进/视功能类诉求时，停止购物建议，只解释风险并建议线下专业处理。
2. **Red-conservative-shopping**：成人普通单光、没有症状红旗，但来源只是电脑验光或字段明显偏弱时，可以给保守购物方向。

`Red-strict` 只做这些事：
- 解释为什么当前数据还不能直接决策
- 列出缺失或薄弱的验光项目
- 只给原则级镜框建议或转诊建议

`Red-conservative-shopping` 可以额外给：
- 小而稳的镜框约束
- 折射率范围
- 普通单光 / 中端升级档位
- 必须明确说明：这不是最终处方级推荐，不能据此默认上高端定制

### 第三步：推导镜框约束

这是核心输出，不要把它简化成一句“选小框”。

始终要输出：
- 目标 `FPD` 范围
- `FPD` 和移心量的计算公式；如果用户给了镜框，必须代入计算
- 建议片高上限
- 片高建议对应的屈光负担估算公式
- 框型建议：全框 / 半框 / 无框
- 鼻托与稳定性建议
- 是更该优先控宽，还是更该优先控高

如果用户已经选了镜框，先判断镜框适配性，再谈镜片升级。

镜框推荐必须展示计算口径，不要只给“选小框”结论：
- `FPD = A + DBL`
- 只有总 `PD` 时：`单眼移心量 ≈ abs(FPD - 总PD) / 2`
- 有单眼 `PD` 时：`右眼移心量 = abs(FPD / 2 - 右单眼PD)`，`左眼移心量 = abs(FPD / 2 - 左单眼PD)`
- 反推目标 `FPD` 时：
  - 优秀目标：`目标FPD <= 总PD + 3 mm`
  - 实用目标：`目标FPD <= 总PD + 6 mm`
  - 通常避免：`目标FPD > 总PD + 10 mm`
- 如果用户给的是单眼 `PD`，可先用 `总PD = 右单眼PD + 左单眼PD` 估算购物目标，再分别检查左右移心量
- 对称估算有局限：脸型、鼻梁、镜架佩戴位置和加工中心可能影响实际结果，最终以试戴和加工确认为准

用这些经验阈值：
- 在很多场景里，小而居中的镜框，往往优于大框配高折射率
- 单眼移心量 `<= 1.5 mm` 很优秀
- `1.5 - 3 mm` 较好
- `3 - 5 mm` 临界，要结合度数和适应能力看
- `> 5 mm` 风险高，尤其是高度近视或高散光

片高压力这样判断：
- 粗略屈光负担用每眼 `max(abs(SPH), abs(SPH + CYL))` 估算；两眼取更高值
- 联合光度很高（约 `>= 8.00D`）：优先 `30 mm` 左右
- 中高联合光度（约 `6.00D - 8.00D`）：优先 `<= 35 mm`
- 中等或更低负担：通常 `<= 40 mm`
- 这个估算只用于镜框和厚薄压力，不替代正式处方

如果用户高散光：
- 轴位接近 `90°`：更要严格控宽
- 轴位接近 `180°`：更要严格控高

尊重已经稳定的适应：
- 如果用户多年佩戴并不理想的镜框但没有明显不适，不要强行大改
- 更适合做保守、渐进式优化

### 第四步：评估现有镜框

如果用户给了镜框参数，输出这些内容：
- 镜框尺寸
- `FPD`
- 左右眼估算移心量，并展示代入公式
- 适配结论：优秀 / 可接受 / 临界 / 不建议
- 主要风险：过宽 / 过高 / 不稳 / 鼻托支撑差
- 最终建议：保留 / 谨慎保留 / 换更小框

### 第五步：推荐折射率

不要把“多少度对应多少折射率”当死表去背。推荐时至少考虑：
- 屈光负担
- 镜框大小和移心量
- 对厚薄的在意程度
- 预算

默认实用逻辑：
- 低负担（约 `< 3.00D`）：`1.50` 或 `1.60`
- 中等负担（约 `3.00D - 5.50D`）：通常 `1.60`
- 中高负担（约 `5.50D - 8.00D`）：通常 `1.71`
- 很高负担（约 `>= 8.00D`）：通常 `1.74`

调档规则：
- 镜框小、移心量小、片高低：可优先保守折射率
- 镜框偏大、片高高、用户很在意薄：可上调一档
- 预算敏感或更在意成像舒适：不要盲目上调
- 这些阈值是购物建议，不是医学或加工硬标准

优先考虑：
- `1.50`
- `1.60`
- `1.71`
- `1.74`

谨慎或弱化推荐：
- `1.56`
- `1.67`

解释权衡，不要只是报数字。

### 第六步：先定镜片档位，再给品牌示例

先选档位：

1. 普通单光实用型
2. 中端升级型
3. 高端定制单光型

只有满足这些条件才进入高端定制：
- 当前案例是 `Green`
- 镜框稳定性好
- 定位数据足够强
- 用户理解高端镜片效果依赖验配质量
- 明确有瞳高或门店能按所选镜框完成可靠定位

如果定位数据弱：
- 不要把蔡司这类定制产品当默认答案主动推

如果用户问“蔡司泽锐 / 智锐值不值”：
- 先强调它们是建立在更精确的瞳距、瞳高、镜架定位数据上的
- 再解释它们偏向近距离优化，不是无条件全面更清楚
- 需要时读取 [references/linked-sources.md](references/linked-sources.md) 的高端镜片部分

如果用户继续追问“那是不是预算够就直接上高端定制”：
- 不要直接回答“是”
- 先回到这几个条件：
  - 当前是否真的是 `Green`
  - 镜框是否稳定
  - 是否有瞳高和可靠定位数据
  - 用户主要诉求是不是近中距离优化
- 如果这些条件不够，就把高端定制降级成“可讨论选项”，不是默认答案

### 第七步：输出结构化报告

始终按“配镜决策报告”来输出：
- 数据可信度
- 字段完整度和处方可信度的关键依据
- 处方解读
- 镜框约束或现有镜框评估
- 折射率建议
- 镜片档位和品牌示例
- 当前不建议项
- 简短的下单前检查清单

需要时直接使用 [references/selection-reference.md](references/selection-reference.md) 里的模板。

如果需要更贴近真实对话的完整成品，优先使用 [references/conversation-templates.md](references/conversation-templates.md) 的输出示例。

如果用户追问“为什么这样推荐”，优先从这些维度解释：
- 镜框几何约束
- 验光质量下限
- 高端镜片对定位数据的依赖
- 蓝光和数码眼疲劳的证据强弱

如果用户追问“防蓝光有没有用”“我天天看电脑是不是必须防蓝光”：
- 先明确：不把防蓝光当默认必选项
- 解释重点放在：
  - 蓝光未被证明会伤害眼睛
  - 数码眼疲劳更常见的原因通常不是蓝光本身
  - 更优先检查验光是否准确、镜框是否合适、屏幕距离和用眼习惯
- 如果用户坚持，再把防蓝光作为偏好项，而不是护眼刚需

## 决策规则

- 先解决镜框问题，再谈镜片升级
- 不要把这个 skill 当医疗工具
- 数据弱时，不要把话说得太满
- 明确解释推荐理由
- 优先输出可执行的购物约束，而不是大段理论

## 常见停机条件

出现以下情况时，不要继续给精确高端方案：
- 只有电脑验光
- 轴位缺失
- 多次验光结果明显冲突
- 有明显双眼视异常相关抱怨
- 用户要求渐进 / 防控 / 功能型产品
- 想做高端定制，但没有可靠定位数据

## 输出风格

- 先给结论
- 表达务实、直接
- 多用明确范围、阈值和权衡
- 不要把回答变成价目表

## 来源使用规则

- 把官方健康类材料优先用于“蓝光 / 数码眼疲劳 / 屏幕用眼建议”
- 把经验型文章优先用于“镜架三姿态 / 瞳高 / 高端镜片验配 caveat”
- 如果链接正文无法直接访问，不要把无法核验的细节写成硬规则
- 可以保留这些链接作为延伸阅读，但 skill 的强规则应只建立在已稳定抽取出的结论上
