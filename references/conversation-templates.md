## 1. 标准问诊开场

适合用户一上来就说“帮我配眼镜”“推荐镜片”的场景。

```text
先别急着选品牌，我先帮你把配镜信息收完整，再给你一个更稳妥的建议。

先发我这几项：
1. 右眼和左眼度数（如果有散光，把散光和轴位一起发）
2. 瞳距 PD（有单眼瞳距更好）
3. 你主要是日常通勤、电脑办公，还是看远/开车更多
4. 有没有已经看中的镜框，或者你更想要大框还是小框
```

## 2. 两条输入路径

### 路径 A：文字输入

适合用户直接发：
- 右眼、左眼度数
- `PD`
- 使用场景
- 镜框参数

建议开场：

```text
你直接把右眼、左眼度数和 PD 发我就行，有散光的话把轴位一起发，我按文字处方直接帮你走配镜判断。
```

### 路径 B：图片上传

适合用户上传：
- 验光单照片
- 医院打印单
- 眼镜店处方单
- 电脑验光单

建议开场：

```text
你可以直接把验光单图片发我，我先帮你把里面的度数、散光、轴位和 PD 提取出来，再和你确认关键字段，确认后再给你镜框和镜片建议。
```

### 图片路径确认话术

```text
我先把图片里能识别到的信息整理出来，但有几个字段还需要你确认一下，特别是散光轴位和 PD。这里的“确认”只是确认我有没有读对图片，不代表这张单子自动等于最终配镜处方。确认后我再继续给你镜框和镜片建议。
```

## 3. 缺信息时的追问顺序

信息不全时，不要一次问太多，按优先级补。

### 用户只给了近视度数

优先追问：

1. 散光和轴位
2. `PD`
3. 使用场景
4. 是否已有镜框
5. 旧镜是否舒适

建议话术：

```text
你现在这组数据还不够我直接推荐镜片，至少还差这几个关键项：
1. 有没有散光，散光轴位是多少
2. 瞳距 PD
3. 主要是看电脑多，还是看远多
4. 有没有已经选中的镜框
```

### 用户有完整处方，但没给镜框

优先追问：

1. `PD`
2. 预算
3. 是否在意厚薄
4. 想要大框还是小框
5. 旧镜体验

### 用户有完整处方，也有镜框

优先追问：

1. `A / DBL / 片高`
2. `PD`
3. 是否稳定、压鼻、下滑
4. 预算和是否在意厚薄

## 4. 快速问诊模板

### 模板 A：轻量版

适合普通用户、聊天场景。

```text
我先收 4 个关键信息，再给你推荐：

1. 右眼和左眼度数（有散光的话把散光和轴位一起发）
2. 瞳距 PD
3. 你主要是看电脑多，还是看远/开车多
4. 有没有已经选中的镜框，或者你偏好大框还是小框
```

### 模板 B：完整配镜版

适合用户明确要一次性拿完整建议。

```text
为了把建议给准一点，你按这个格式发我：

- 右眼 OD：SPH / CYL x AXIS
- 左眼 OS：SPH / CYL x AXIS
- PD：总瞳距或单眼瞳距
- 用途：日常 / 电脑 / 看远 / 开车
- 预算：大概区间
- 是否在意厚薄：在意 / 一般 / 不在意
- 旧镜体验：舒适 / 压鼻 / 头晕 / 边缘变形 / 容易下滑
- 镜框信息（如果有）：A、DBL、片高、镜腿长
```

### 模板 C：图片上传路径版

适合用户一开始就发验光单图片。

```text
我先帮你从图片里提取关键信息，优先看这些：
1. 右眼和左眼的度数
2. 散光和轴位
3. PD
4. 这张单子是电脑验光，还是最终配镜处方

我先整理出识别结果给你确认，确认后再继续推荐镜框和镜片。
```

## 7. `Red` 输出模板

适合只有电脑验光、轴位缺失、明显视功能异常等情况。

```markdown

- 数据可信度：`Red`
- 字段完整度：
- 处方可信度：
- 结论：当前数据不足以直接推荐具体镜片型号，先补验光或复核

## 为什么现在不能直接推荐
- 当前问题 1：
- 当前问题 2：
- 当前问题 3：

## 你下一步该补什么
- [ ] 完整主观验光结果
- [ ] 散光轴位
- [ ] PD
- [ ] 必要时瞳高
- [ ] 旧镜试戴反馈

## 现在只能先给的原则
- 镜框先尽量小而稳
- 暂不追求高端定制片
- 先把验光质量和镜框方向做对

## 如果你现在就想先看购物方向
- 只先看小而稳的镜框范围
- 只先看折射率范围和普通单光档位
- 前提是成人普通单光、没有复视/明显视疲劳/眼病/术后等红旗
- 如果有症状红旗或专项验配诉求，应停止购物建议，先去线下专业处理
- 不把这一步当成最终处方级定案
```

## 9. 完整示例：已有镜框参数时

### 用户输入

```text
右眼 500，左眼 450 加 50 散光，PD 64。看中一个 50□19，片高 42 的框，可以吗？
```

### 建议输出

```markdown
## 配镜建议摘要
- 数据可信度：`Yellow`
- 字段完整度：缺散光轴位
- 处方可信度：来源未说明，只能做普通单光保守判断
- 结论：这副框偏大偏高，不是最优，尤其片高 42 对你这组度数不算友好

## 验光数据解读
- 右眼：近视 500 度
- 左眼：近视 450 度 + 散光 50 度
- PD：64
- 风险点：散光轴位未知，无法继续细化“控宽还是控高”

## 镜框评估
- 镜框参数：`50□19`
- 框心距：`69`
- 单眼移心量约：`(69 - 64) / 2 = 2.5 mm`
- 评价：横向还可以，但片高 42 偏高
- 主要问题：框偏高，后续成镜厚度和边缘保留区域都不算理想

## 镜框参数建议
- 目标框心距：优先控制在 `67-70` 以内
- 建议片高上限：优先 `<= 40 mm`，更理想是 `35-38 mm`
- 建议框型：小一点的全框、可调鼻托

## 镜片建议
- 首选折射率：`1.60`
- 如果你坚持这副框或更在意薄：`1.71`
- 示例方向：明月 PMC、凯米 U2、依视路中端单光

## 当前不建议项
- 不建议选更高更大的网红大框
- 不建议在当前数据条件下直接做高端定制
```

- 确认当前处方是否准确
- 确认旧镜有没有偏大、下滑、压鼻或清晰区不对的问题
- 看屏时保持一臂左右距离，视线略向下
