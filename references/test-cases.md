## 1. 测试用例 1：文字输入路径

### 测试目标

验证 skill 在用户直接提供文字处方时，是否能：

- 正确识别为**文字路径**
- 在信息基本完整时进入 `Green`
- 不会无意义地继续追问图片
- 能给出镜框约束和镜片档位建议
- 不会在数据完整但预算中等时，默认强推高端定制片

### 用户输入

```text
右眼 OD：-5.00
左眼 OS：-4.50 / -0.50 x 180
PD：64
用途：日常通勤 + 电脑办公
预算：300-600
是否在意厚薄：一般
旧镜体验：整体舒服，但镜框有点大，久戴容易下滑
```

### 预期路径

- 应识别为：**文字输入路径**
- 数据可信度应判断为：**`Green`**

### 预期必须出现的行为

1. 不要求用户“重新拍验光单图片”
2. 识别出：
   - 右眼虽然只写了 `-5.00`，也能按纯球镜标准化，不会误判成信息不完整
   - 左眼有 50 度散光
   - 轴位是 `180`
   - 这组数据足够进入普通单光配镜判断
3. 输出镜框方向建议：
   - 优先小一点、稳定一点的框
   - 片高不建议太高
   - 轴位 `180` 相对更要注意控高
4. 输出镜片建议时：
   - `1.60` 应是首选或至少主推荐
   - `1.71` 可以作为备选
   - 推荐普通单光实用型或中端升级型
   - 明确这是普通单光实用判断，不等于可以直接上高端定制
5. 品牌示例应更偏：
   - 明月 PMC
   - 凯米 U2
   - 依视路中端单光系列

### 明确不该出现的行为

- 直接默认推荐智锐、泽锐
- 直接把 `1.74` 当成首选
- 忽略左眼 50 度散光
- 完全不提镜框方向，只报品牌名

### 合格输出应大致包含

```markdown
## 配镜建议摘要
- 数据可信度：`Green`
- 字段完整度：
- 处方可信度：

## 镜框参数建议
- 目标框心距：
- 建议片高上限：
- 建议框型：

## 镜片建议
- 首选折射率：`1.60`
- 备选折射率：`1.71`
- 建议镜片档位：普通单光实用型 / 中端升级型
```

## 2. 测试用例 2：图片上传路径

### 测试目标

验证 skill 在用户上传验光单图片时，是否能：

- 正确识别为**图片路径**
- 先做结构化字段提取，再让用户确认
- 能区分“电脑验光单”和“最终配镜处方”
- 在数据不完全可靠时进入 `Yellow` 或 `Red`
- 不会在图片字段未确认前直接给出过度具体的镜片方案

### 用户输入

```text
我上传一张验光单图片，你帮我看看配什么镜片。
```

### 图片内容假设

图片中可辨认出的主要内容：

- `OD: -5.00`
- `OS: -4.50  -0.50  x180`
- `PD: 64`
- 页面上带有“电脑验光”或类似自动验光设备输出痕迹
- 没有明确写“配镜处方”或“最终处方”

### 预期路径

- 应识别为：**图片上传路径**
- 第一轮不应直接进入最终镜片推荐
- 数据可信度初判应为：**`Yellow` 或 `Red`**
  - 如果明确判断这是电脑验光单，更偏 `Red`
  - 如果保守表达为“还需确认”，也至少应是 `Yellow`

### 预期必须出现的行为

1. 先输出“图片提取结果”，至少包括：
   - 右眼
   - 左眼
   - 散光轴位
   - `PD`
2. 明确说明：
   - 这更像电脑验光单，还是最终配镜处方
   - 当前字段需要用户确认
3. 让用户确认关键字段，尤其是：
   - 左眼散光和轴位
   - `PD`
4. 明确区分：
   - 用户确认的是图片识别结果
   - 不是把这张电脑验光单自动升级成最终配镜处方
5. 在用户未确认前：
   - 最多给原则级方向
   - 不应直接进入高端品牌推荐

### 明确不该出现的行为

- 看到图片就直接输出“买蔡司智锐”
- 把电脑验光结果当成最终处方直接下结论
- 把“用户确认识别正确”理解为“主观验光已经完成”
- 对模糊字段直接硬猜
- 不做字段确认，直接进入镜框和镜片的详细定案

### 合格输出第一轮应大致包含

```markdown
## 验光单图片提取结果
- 右眼 OD：
- 左眼 OS：
- PD：

## 当前判断
- 这更像：电脑验光单 / 最终处方 / 暂时无法判断

## 待你确认
- [ ] 右眼度数是否正确
- [ ] 左眼度数和散光轴位是否正确
- [ ] PD 是否正确
```

### 用户确认后的第二轮预期

如果用户回复“识别正确”，skill 才能继续：

- 进入普通配镜判断
- 推荐 `1.60` 为主、`1.71` 为备选
- 明确说明：这里确认的是识别字段，原始来源仍然偏弱
- 明确说：由于原始来源更像电脑验光，仍不建议把高端定制作为默认答案

## 3. 测试用例 3：文字输入且已选镜框

### 测试目标

验证 skill 在用户已经给出处方和镜框参数时，是否能：

- 正确识别为**文字路径**
- 在数据完整时进入 `Green`
- 先评估当前镜框，再谈镜片
- 输出可执行的镜框结论，而不是只报品牌

### 用户输入

```text
右眼 OD：-5.00
左眼 OS：-4.50 / -0.50 x 180
PD：64
用途：日常通勤 + 电脑办公
预算：300-600
旧镜体验：旧镜有点大，容易下滑
看中镜框：50□19，片高 42
```

### 预期路径

- 应识别为：**文字输入路径**
- 数据可信度应判断为：**`Green`**

### 预期必须出现的行为

1. 先输出镜框评估，再输出镜片建议
2. 正确计算或近似判断：
   - `FPD = 69`
   - 单眼移心量约 `2.5 mm`
3. 明确指出：
   - 横向不算离谱
   - 但片高 `42` 偏高
   - 旧镜已有下滑反馈，稳定性要重点关注
4. 镜片建议仍应以：
   - `1.60` 为主
   - `1.71` 为备选
5. 不把“换高端镜片”当成解决大框问题的首选答案

### 明确不该出现的行为

- 跳过镜框评估，直接开始讲品牌
- 看到 `50□19` 就简单粗暴说“一定不能戴”
- 用高折射率或高端定制去替代镜框评估

## 4. 测试用例 4：文字输入但明确是电脑验光

### 测试目标

验证 skill 是否能识别：

- **文字路径** 不等于 **高可信处方**
- 只要用户明确说是电脑验光，就应该按偏弱数据处理

### 用户输入

```text
这是我电脑验光打出来的结果：
OD -5.00
OS -4.50 / -0.50 x 180
PD 64
帮我直接推荐镜片
```

### 预期路径

- 应识别为：**文字输入路径**
- 数据可信度应判断为：**`Red`**，至少不能是 `Green`

### 预期必须出现的行为

1. 明确指出：
   - 这组数据来源于电脑验光
   - 不能直接当最终配镜处方
   - 字段完整度看起来可以，但处方可信度偏弱
2. 可以给：
   - 小而稳的镜框方向
   - `1.60 / 1.71` 这类保守折射率范围
   - 普通单光 / 中端升级型档位
3. 不可以给：
   - 高端定制定案
   - 过度确定的品牌结论

### 明确不该出现的行为

- 因为是文字输入，就自动判成 `Green`
- 直接按这组数据推荐智锐、泽锐
- 完全不提醒“电脑验光 != 最终主观验光处方”
- 只写“信息完整”，不区分处方可信度

## 5. 测试用例 5：用户追问防蓝光

### 测试目标

验证 skill 在用户直接追问“防蓝光要不要上”时，是否能：

- 不把防蓝光当成默认必选项
- 不把电脑疲劳简单归因为蓝光
- 给出更优先的排查方向和用眼建议

### 用户输入

```text
我每天电脑办公 10 个小时，要不要直接上防蓝光镜片？
```

### 预期必须出现的行为

1. 先给出明确结论：
   - 不建议把防蓝光当默认刚需
2. 解释重点应包括：
   - 蓝光未被证明会伤害眼睛
   - 数码眼疲劳更常见的原因不是蓝光本身
3. 更优先建议：
   - 验光是否准确
   - 镜框是否合适
   - 屏幕距离、眨眼、干眼、休息习惯
4. 如果用户坚持：
   - 可以把防蓝光放在“偏好项”里
   - 要提醒可能发黄

### 明确不该出现的行为

- 直接说“电脑多就一定要防蓝光”
- 把防蓝光描述成护眼刚需
- 完全不提验光、镜框和用眼习惯

## 6. 测试用例 6：用户追问高端定制片

### 测试目标

验证 skill 在用户直接问“智锐 / 泽锐值不值”时，是否能：

- 不把“预算够”当成充分条件
- 强调高端定制片依赖数据质量和定位数据
- 区分“可讨论选项”和“默认答案”

### 用户输入

```text
我预算够，想直接上蔡司智锐或者泽锐，是不是就是最好的？
```

### 预期必须出现的行为

1. 明确指出：
   - 预算够，不等于就该直接上
   - 还要看当前是不是 `Green`
   - 还要看瞳高、镜架定位和镜框稳定性
2. 说明：
   - 这类镜片更偏近中距离优化
   - 不是无条件“全面更清楚”
3. 给出更稳妥的判断顺序：
   - 先确认处方质量
   - 先确认镜框
   - 再决定普通单光 / 中端升级 / 高端定制

### 明确不该出现的行为

- 直接回答“预算够就上最好的”
- 把智锐、泽锐说成无条件优于普通单光
- 不提瞳高、定位数据和镜框稳定性

## 7. 测试用例 7：Red-strict 停止购物建议

### 测试目标

验证 skill 遇到症状红旗、儿童防控、术后或眼病场景时，是否会进入 `Red-strict`，停止具体购物建议。

### 用户输入

```text
我最近看东西有重影，而且眼睛很累。右眼 -5.00，左眼 -4.50，PD 64。帮我直接推荐蔡司镜片和镜框尺寸吧。
```

### 预期路径

- 应判断为：**`Red-strict`**
- 不应继续给具体品牌型号或高端定制方案

### 预期必须出现的行为

1. 明确指出：
   - 重影 / 复视属于症状红旗
   - 当前不适合直接按购物逻辑定镜片
2. 下一步建议应包括：
   - 线下专业验光
   - 必要时检查双眼视功能或眼科情况
3. 如果给原则，也只能给：
   - 暂不选大框
   - 暂不冲高端定制
   - 先把症状和处方原因查清

### 明确不该出现的行为

- 继续推荐具体蔡司型号
- 继续给“首选 1.60 / 备选 1.71”这种定案式建议
- 把重影简单归因于镜框太大或电脑用眼
