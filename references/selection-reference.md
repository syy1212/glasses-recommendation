# 配镜参考

## 1. 输入标准化

### 最低可用验光字段

- `OD`: `SPH / CYL x AXIS`
- `OS`: `SPH / CYL x AXIS`
- `PD`：优先单眼瞳距；没有时接受总瞳距

### 纯球镜标准化

- 如果某只眼只有球镜度数，且没有散光记录，可先标准化为 `CYL 0.00`
- 只有在存在散光时，`AXIS` 才是硬必填项
- 纯球镜没有 `AXIS`，不应自动判定为信息不完整
- 文字路径里，如果同时有 `PD`、用途和旧镜反馈，普通单光场景通常已可进入实用判断

### 建议补充字段

- 当前眼镜的舒适度和适应情况
- 镜框参数：`A`、`DBL`、镜腿长、片高
- 验光质量线索：
  - 红绿测试
  - JCC / 交叉圆柱镜精校
  - 试戴
  - 瞳高测量

### 红旗输入模式

- 只有“电脑验光”结果，没有主观确认
- 散光有意义但轴位缺失
- 多次验光结果冲突很大
- 严重视疲劳、重影、复视、融像异常抱怨
- 明显屈光参差但没有适应史

### 入口路径与可信度的关系

- 文字输入路径，不等于高可信处方
- 只要用户明确说“这是电脑验光结果”“机器测的”，即使字段齐全，也不能自动视为最终配镜处方
- 这类场景通常应判为偏保守的 `Yellow` 或直接 `Red`
- 后续可以给：
  - 小而稳的镜框方向
  - 折射率范围
  - 普通单光 / 中端升级档位
- 不应直接给：
  - 高端定制片定案
  - 过度自信的品牌结论

### 字段完整度与处方可信度

分级前先拆开两个维度：

- **字段完整度**：`SPH / CYL / AXIS / PD / 用途 / 镜框参数` 是否够用
- **处方可信度**：是否有主观验光、试戴、红绿测试、JCC、旧镜适应反馈；是否只是电脑验光

常见组合：

- 字段完整 + 来源可靠：可进入普通单光 `Green`
- 字段完整 + 明确只是电脑验光：不能进入 `Green`，通常按 `Red-conservative-shopping`
- 字段略少 + 来源可靠 + 旧镜稳定：可给保守普通单光建议，避免高端定制
- 有症状红旗、儿童防控、眼病/术后、渐进或视功能类诉求：按 `Red-strict`

高端定制判断要单独加门槛：
- 处方可信度高
- 镜框稳定
- 有瞳高或可靠定位数据
- 用户理解收益依赖验配质量

## 2. 图片验光单提取规则

### 适用图片类型

- 眼镜店验光处方单
- 医院验光结果单
- 电脑验光结果单
- 手写或打印的配镜处方单

### 先提取这些字段

优先顺序：

1. 右眼 `OD`
- `SPH`
- `CYL`
- `AXIS`

2. 左眼 `OS`
- `SPH`
- `CYL`
- `AXIS`

3. `PD`
- 总瞳距
- 单眼瞳距（如果有）

4. 其他可选字段
- 瞳高
- `ADD`
- 验光日期
- 备注中提到的用途或试戴意见

### 提取时的注意事项

- 先区分这张图片更像：
  - **电脑验光结果**
  - **主观验光 / 最终配镜处方**
- 如果同一张单子上同时有多组数字，优先找：
  - 标注为“处方”“配镜处方”“最终结果”的字段
- 如果只看到电脑验光值：
  - 可以提取
  - 但必须标记为“仅电脑验光，不能直接视为最终配镜处方”

### 遇到这些情况不能硬猜

- 图片模糊
- 手写字段看不清
- 散光轴位残缺
- 左右眼列错位
- 同一字段有两个候选值

### 图片路径的工作要求

- 先把提取结果结构化列出来
- 明确告诉用户哪些字段是“已识别”，哪些是“待确认”
- 让用户确认后，再进入镜框和镜片推荐
- 用户确认的是“字段识别结果”，不是把电脑验光单升级成最终配镜处方
- 如果来源更像电脑验光单，确认后也只能进入保守建议：
  - 镜框约束
  - 折射率范围
  - 普通单光 / 中端升级档位
  - 不要直接给高端定制定案

### 推荐的提取输出格式

```markdown
## 验光单图片提取结果
- 右眼 OD：`SPH __ / CYL __ x AXIS __`
- 左眼 OS：`SPH __ / CYL __ x AXIS __`
- PD：`__`
- 其他字段：`__`

## 当前判断
- 这更像：电脑验光单 / 最终配镜处方 / 暂时无法判断

## 待你确认
- [ ] 右眼度数是否识别正确
- [ ] 左眼度数是否识别正确
- [ ] 散光轴位是否识别正确
- [ ] PD 是否识别正确
```

## 3. 镜框公式与经验阈值

### 基本公式

- `FPD = A + DBL`
- 如果只有总瞳距：
  - `单眼移心量 ≈ abs(FPD - 总PD) / 2`
- 如果有单眼瞳距：
  - `右眼移心量 = abs(FPD / 2 - 右单眼PD)`
  - `左眼移心量 = abs(FPD / 2 - 左单眼PD)`

### 移心量判断

- `<= 1.5 mm`：优秀
- `1.5 - 3 mm`：较好
- `3 - 5 mm`：临界，但有时还能接受
- `> 5 mm`：风险较高，尤其是高度近视或高散光

### 实用 `FPD` 购物目标

- 优秀目标：`FPD <= 总PD + 3 mm`
- 实用目标：`FPD <= 总PD + 6 mm`
- 通常应避免：`FPD > 总PD + 10 mm`

### 片高建议

- 粗略屈光负担：每眼用 `max(abs(SPH), abs(SPH + CYL))` 估算，两眼取更高值
- 这个估算只用于镜框、片高和厚薄压力，不替代正式处方
- 约 `>= 8.00D`：优先 `30 mm` 左右
- 约 `6.00D - 8.00D`：优先 `<= 35 mm`
- 约 `< 6.00D`：通常 `<= 40 mm`

### 框型建议

- 高度近视：优先全框
- 高度近视：不要默认无框或半框
- 稳定性存疑时，优先可调鼻托
- 比起装饰性大框，优先稳固贴合

### 散光轴位建议

- 轴位接近 `90°`：控宽更重要
- 轴位接近 `180°`：控高更重要

## 4. 折射率逻辑

折射率要结合屈光负担和镜框适配来看，不要只看度数。

### 实用默认逻辑

- 低负担（约 `< 3.00D`）：`1.50` 或 `1.60`
- 中等负担（约 `3.00D - 5.50D`）：通常 `1.60`
- 中高负担（约 `5.50D - 8.00D`）：通常 `1.71`
- 很高负担（约 `>= 8.00D`）：通常 `1.74`

### 调档因素

- 小框、低片高、移心量小：可优先低一档或保守折射率
- 大框、高片高、移心量临界、用户很在意薄：可上调一档
- 预算敏感、在意成像舒适、阿贝数敏感：不要盲目上调
- 数据可信度弱时，只给折射率范围，不给最终型号定案

### 优先考虑的折射率

- `1.50`
- `1.60`
- `1.71`
- `1.74`

### 弱化推荐的折射率

- `1.56`
- `1.67`

原因：
- 它们在实际体验上的收益，往往不足以抵消带来的妥协

## 5. 镜片档位示例

### 档位 1：普通单光实用型

适合：
- `Yellow` 案例
- 预算优先的用户
- 没有可靠瞳高数据的用户
- 只想配一副稳妥单光镜的用户

示例方向：
- 万新
- 明月 PMC
- 凯米 U2
- 依视路基础单光系列

### 档位 2：中端升级型

适合：
- `Green` 或较强的 `Yellow` 案例
- 想要更通透、膜层更好的日常镜片用户
- 中等预算用户

示例方向：
- 明月较高性价比单光系列
- 依视路中端系列
- 蔡司标准单光系列

### 档位 3：高端定制型

适合：
- `Green` 案例
- 镜框稳定
- 定位数据强
- 有瞳高或可按所选镜框完成可靠定位
- 理解高端镜片依赖验配质量的用户

示例方向：
- 蔡司 SmartLife 风格的定制单光产品
- 同级别高端定制单光产品

出现以下情况时不要主动推这个档位：
- 缺少瞳高
- PD 质量偏弱
- 镜框本身不稳定

## 6. 特殊需求处理

### 防蓝光

默认态度：
- 不主动推荐

如果用户坚持：
- 把它定义为偏好选择，而不是硬性的护眼刚需
- 提醒可能发黄

### 防污抗油

适合：
- 厨房环境
- 多灰尘环境
- 非常在意膜层维护的用户

### 长时间电脑使用

不要默认把所有电脑用户都往高端近用优化镜片上推。

更好的处理方式：
- 先判断用户是不是其实只需要普通单光镜
- 如果用户明确追求近中距离优化，要强调它更依赖精确验配

## 8. 沟通规则

- 先给主结论
- 解释权衡，而不是只报答案
- 不要比数据本身更自信
- 优先讲镜框约束和购买决策，少堆长篇光学理论
