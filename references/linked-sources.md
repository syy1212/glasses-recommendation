### 可稳定使用的支撑点

镜架三姿态相关内容主要用来支撑这些 skill 结论：

- 前倾角、镜眼距、面弯会影响佩戴舒适性和视觉质量
- 这些参数通常属于更进阶的验配内容，普通门店未必会测
- 常见普通镜框可参考的经验范围大致是：
  - 前倾角 `8-15°`
  - 镜面角 `0-10°`
  - 镜眼距 `10-15 mm`
- 这些参数不是普通单光配镜的第一优先级
- 它们更适合作为：
  - 高端定制镜片的附加定位数据
  - 用户追问“为什么高级镜片要测这些”的解释材料

### skill 应如何使用

- 不把三姿态写成普通用户的强制收集字段
- 在“高端镜片 / 近距离优化 / 用户主动追问”时再展开
- 默认仍以：度数、散光、`PD`、镜框几何、瞳高为更高优先级

## 3. 验光流程与点瞳高

### 相关链接

- 验光流程参考：<https://zhuanlan.zhihu.com/p/465598173>
- 单光是否需要瞳高：<https://www.zhihu.com/question/419831439/answer/2982129907>

### 可稳定使用的支撑点

- 判断验光质量的最低下限，可以抓这几个信号：
  - 红绿测试
  - 交叉圆柱镜 / `JCC`
  - 有一定试戴
- 点瞳高在普通单光里未必是绝对必需，但：
  - 能点尽量点
  - 点瞳高没有明显坏处
  - 点瞳高前应先把镜框调整到稳定、习惯的佩戴位置
- 单光镜片里，瞳高更像“进一步提升定位准确性”的手段
- 对高端镜片，瞳高的重要性明显上升

### skill 应如何使用

- 在数据闸门里，把 `JCC / 红绿 / 试戴 / 瞳高` 作为验光质量线索
- 普通单光在 `Yellow` 条件下可以没有瞳高
- 高端定制片、自由曲面、近距离优化类产品，没有瞳高时不要积极推荐

## 4. 高端镜片验配：泽锐 / 智锐

### 相关链接

- 泽锐镜片是否有问题：<https://www.zhihu.com/question/565754223/answer/2762543447>
- 配泽锐验光注意事项：<https://www.zhihu.com/question/532913125/answer/2628352886>
- 瞳高与佩戴位置：<https://www.zhihu.com/question/595031329/answer/2983544430>
- 泽锐看远和边缘模糊：<https://www.zhihu.com/question/553686097/answer/2832223060>

### 可稳定使用的支撑点

- 泽锐、智锐这类镜片的收益依赖更精确的定位数据
- 核心不是“品牌贵”，而是：
  - 瞳距
  - 瞳高
  - 镜架佩戴位置
  - 必要时的三姿态参数
- 经验型文章通常强调：
  - 泽锐对瞳高和验配要求更敏感
  - 智锐相对把一部分处理做进了镜片定位逻辑
- 可采用的经验总结是：
  - 这类镜片更偏近距离优化
  - 不是无条件“更锐更清楚”
  - 可能带来远用清晰度妥协或边缘主观模糊感

### skill 应如何使用

- 高端定制片必须要求更高数据质量
- `Green` 以外不要主动推这类产品
- 如果用户主要诉求是“看远最清楚”，要谨慎推荐近距优化型产品
- 如果用户长期戴普通非球面且很稳定，不要一上来就说“直接升级高端定制”

## 5. 防蓝光与数码眼疲劳

### 相关链接

- AAO：蓝光眼镜是否值得购买  
  <https://www.aao.org/eye-health/tips-prevention/are-computer-glasses-worth-it>
- AAO：数码设备与眼睛  
  <https://www.aao.org/eye-health/tips-prevention/digital-devices-your-eyes>

### 可稳定使用的支撑点

根据 AAO 公开材料，可抽取这些对 skill 有用的强结论：

- 电脑和手机的蓝光**未被证明会伤害眼睛**
- 蓝光眼镜**不是**缓解数码眼疲劳的核心方案
- 改善睡眠也**不需要**额外买蓝光眼镜，减少夜间看屏和开启夜间模式更直接
- 数码眼疲劳更常见的原因包括：
  - 近距离长时间注视
  - 眨眼减少
  - 干眼
  - 眩光
  - 屏幕距离与姿势不佳

### AAO 建议

- `20-20-20` 休息法则
- 眼干时可用人工泪液
- 保持约一臂距离，视线略向下
- 减少眩光和过强亮度
- 长时间看屏时，隐形用户可以改戴框架眼镜

### skill 应如何使用

- 默认不主动推荐防蓝光
- 如果用户坚持，可以当作偏好项，而不是护眼刚需
- 当用户说“看电脑眼睛很累”，优先引导：
  - 验光是否准确
  - 旧镜是否不合适
  - 屏幕使用习惯是否有问题
- 不要把“数码眼疲劳”直接归因为蓝光

## 6. 镜片材料与膜层

### 相关链接

- 镜片主流材料介绍：<https://zhuanlan.zhihu.com/p/78606167>
- 镜片结构与膜层：<https://zhuanlan.zhihu.com/p/68195818>

### 可稳定使用的支撑点

- 镜片材料相关决策，核心围绕：
  - 折射率
  - 阿贝数
  - 厚薄诉求
- 膜层方面，可以把“防尘、防油、防水”拆成不同层次：
  - 防尘主要关联防静电层
  - 防水防油主要关联疏水膜
- 结论不是“没用”，而是：
  - 有效果
  - 但不值得夸大成决定性差异

### skill 应如何使用

- 当用户追问“膜层有没有必要”时，回答可以更细：
  - 厨房、灰尘、油污环境更值得在意膜层
  - 普通场景不要把膜层神化

## 8. 给 skill 的可执行结论

这些是从外链体系中，最值得沉淀进 skill 的部分：

1. **把镜架三姿态降级为进阶参数**
- 普通单光主流程不强制收
- 高端镜片和主动追问时再展开

2. **把红绿 / JCC / 试戴 / 瞳高纳入验光质量闸门**
- 它们不直接决定品牌
- 但决定是否允许推荐高端方案

3. **把泽锐 / 智锐写成“高依赖验配质量”的产品**
- 不是默认最优
- 需要更完整的数据才能推荐

4. **把防蓝光从“默认推荐项”移出**
- 默认不主动推荐
- 用户坚持时作为偏好项处理

5. **把数码眼疲劳处理成用眼方式问题优先**
- 先查处方、镜框和用眼习惯
- 再谈特殊镜片
